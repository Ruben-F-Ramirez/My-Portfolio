Floating-Point Processing Unit (FPU)

Project Overview

This project involves designing and implementing a Floating-Point Processing Unit (FPU) using Verilog. The FPU is modular and supports multiple cores for parallel processing, performing floating-point arithmetic operations such as addition, subtraction, and multiplication.

Key Features

  • Modular architecture supporting multi-core processing.
  • Floating-point arithmetic operations: Addition, subtraction, and multiplication.
  • Dedicated shift registers for efficient pipelining.
  • Integrated timing control for managing enable and valid signals.
  • Supports sequential and parallel execution models.

Design Details

The FPU consists of multiple key modules, each responsible for a specific task in floating-point computation:

  • FP_adder_13ccs: A 13-cycle floating-point adder.
  • FP_multiplier_10ccs: A 10-cycle floating-point multiplier.
  • FP_subtractor_13ccs: A 13-cycle floating-point subtractor.
  • FP_datapath: Integrates the ALU components to process floating-point operations sequentially.
  • fppu: Instantiates two FPU cores for parallel floating-point processing.
  • quad_core: Instantiates four FPU cores for high-performance operations.
  • shift13_register: A 13-stage shift register used for pipelining.
  • shift23_register: A 23-stage shift register for longer pipeline processing.
  • timing_ctl: Manages enable signals, valid signals, and operation timing.
  • fpu: The top-level module integrating all submodules into a fully functional Floating-Point Processing Unit.

Simulation & Testing

The FPU is tested using simulation tools such as ModelSim and Vivado to verify the correctness of its computations. The testing process includes:

  • Verifying the accuracy of floating-point addition, subtraction, and multiplication.
  • Ensuring correct operation timing and pipeline delays.
  • Validating enable and valid signal management within the timing control module.
  • Confirming proper execution of both single-core and multi-core FPU designs.
